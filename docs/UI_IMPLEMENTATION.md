# Реализация UI (Фотостудия «Экспозиция»)

## 1) Ключевые экраны

1. Главная страница (`/`)  
Минималистичный hero, акцент на центральной CTA-кнопке «Забронировать зал», навигация по разделам.

2. Каталог залов (`/halls`)  
Карточки студий (фото, цена, оборудование), фильтры и сортировка, переход к деталям зала.

3. Личный кабинет и бронирования (`/my-bookings`, `/profile`)  
История заказов, фильтрация по статусу, карточки со статусными бейджами, оплата и отмена.

4. Карточка зала и бронирование (`/halls/:id`)  
Слоты доступности с блокировкой занятых интервалов, форма брони (дата/время), доп.услуги и клиентский расчёт суммы.

5. Экран AI-прогноза (`/ai-insights`)  
Выбор студии + период, запрос к `POST /api/forecast`, тепловая карта загрузки, рекомендации.

6. Админ-панель (`/admin-panel`)  
Вкладки аналитики, залов, заказов, пользователей, логов аудита, услуг и промокодов.

7. Расписание менеджера (`/manager/schedule`)  
Timeline по дням/часам, занятые слоты, боковая панель заказа и смена статуса.

## 2) Навигация

- Публичные: `/`, `/halls`, `/portfolio`, `/login`.
- Авторизованные: `/my-bookings`, `/profile`, `/checkout`, `/ai-insights`.
- Админ/менеджер: `/admin-panel`, `/admin/audit`, `/manager/schedule`.

## 3) UI-решения

- Единый монохромный дизайн и шрифт Garamond/EB Garamond.
- Единый кастомный date-picker на всех экранах вместо системного `input[type=date]`.
- Бейджи статусов заказов и мягкие состояния загрузки/ошибок.
- Heatmap загрузки с цветовой легендой и текстовыми рекомендациями под графиком.
- В админке добавлен отдельный CRUD-блок для доп.услуг, которые используются в форме бронирования.
