# QA чеклист UI (10+ проверок)

## Аутентификация

1. На `/login` регистрация не отправляется при пустом `email`.
2. Пароль короче 8 символов показывает ошибку валидации.
3. Ошибка API входа (неверный логин/пароль) выводится в понятном сообщении.
4. После успешного входа токены появляются в `localStorage` (`access_token`, `refresh_token`).

## Каталог и карточка зала

5. На `/halls` карточка зала показывает фото, цену и минимум 1 элемент оборудования.
6. Фильтры в каталоге (`q`, `capacity`, `price`) меняют результат и сохраняются в URL.
7. На `/halls/:id` занятые слоты визуально недоступны и не кликаются.
8. Клик по свободному слоту автозаполняет время начала/окончания в форме брони.
9. Выбор доп.услуг влияет на клиентский расчёт суммы (база + услуги).
10. Кнопка создания брони недоступна без выбранной даты.

## Бронирования и оплата

11. На `/my-bookings` фильтр по статусу корректно меняет список.
12. Кнопка «Отменить» доступна только для статусов `NEW`/`PENDING`/`CONFIRMED`.
13. Перед отменой открывается подтверждение, после подтверждения статус обновляется.
14. На `/checkout` при успешной оплате появляется success-состояние.

## AI и админка

15. На `/ai-insights` запрос прогноза строит heatmap по выбранному периоду.
16. Под heatmap отображается текстовый блок рекомендаций.
17. На `/admin-panel` вкладка «Пользователи» показывает роли.
18. На `/admin-panel` вкладка «Логи аудита» фильтруется по дате/типу/поиску.
19. На `/admin-panel` вкладка «Услуги» позволяет создать, редактировать и удалить услугу.
20. На `/manager/schedule` клик по занятым слотам открывает боковую панель со сменой статуса.
